## 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 桌面容器 | Tauri v2 (Rust) | 系统原生 WebView，透明窗口，鼠标穿透 |
| Live2D 渲染 | PixiJS 8 + pixi-live2d-display | 模型加载、动画、口型同步 |
| 前端 UI | React 18 + TypeScript + Vite | 聊天气泡、设置面板 |
| 后端内核 | Go PetCore (CGO_ENABLED=0) | AI Agent、记忆管理、插件系统 |
| 记忆存储 | SQLite + FTS5（modernc.org/sqlite） | 结构化记忆 + 全文检索 |
| LLM 协议 | Provider 接口 + 注册表模式 | 统一接入 OpenAI/DeepSeek/Ollama |
| 工具协议 | MCP (Model Context Protocol) | 标准工具扩展 |
| 通信协议 | Tauri IPC (invoke + event) + sidecar stdio | Rust ↔ JS ↔ Go 三层通信 |

## 核心设计原则

| 原则 | 说明 |
|------|------|
| **内核-壳分离** | PetCore (Go) 无 UI 依赖，仅通过 event.Sink 接口输出，可被 Desktop/CLI/Server 多壳复用 |
| **配置驱动** | 所有行为通过 TOML 配置声明，不硬编码 |
| **事件驱动** | 内核发出事件，各 Shell 按需消费 |
| **热加载优先** | L1 YAML 动作包 / L2 JS 脚本文件变更自动重载 |
