# 提交前检查清单

> **每次提交和推送前必须执行**，一条都不能跳过。

## 1. 本地检查

### 前端（修改了 `frontend/` 下的文件时）

```bash
cd frontend
pnpm lint         # 0 errors
pnpm format:check # All matched files use Prettier code style
pnpm typecheck    # 0 errors
```

### Rust（修改了 `src-tauri/` 下的文件时）

```bash
cd src-tauri
cargo clippy -- -D warnings  # 0 warnings
cargo test --all-targets     # all tests pass
```

### Go（修改了 `petcore/` 下的文件时）

```bash
cd petcore
go vet ./...    # 0 errors
go test ./...   # all tests pass
```

## 2. 推送后检查

推送后等待 CI 跑完，确认以下所有项目为绿色：

- [ ] **CI workflow** — 全部 job 通过（含 lint / test / build / coverage）
- [ ] **CodeQL** — `security/code-scanning` 0 个 open alert
- [ ] **Dependabot** — `security/dependabot` 0 个 open alert
- [ ] **auto-release / release** — 这些 workflow 也不应有 failure

## 3. 修复原则

| 原则 | 说明 |
|------|------|
| **全局搜索同类问题** | 修了一个模式，就 grep 全项目确认没有其他同模式问题 |
| **本地和远程一致** | CI 通过的规则本地也必须能跑过 |
| **所有 workflow 都确认** | 不只是 CI，还有 CodeQL / Dependabot / release 等 |
