# UI 开发流程

> 定义 AI agent 在开发桌面端 UI 时，从接到需求到交付完成的完整工作流。
>
> **核心原则：先设计 → 后计划 → 再编码 → 验证 → 交付。禁止一上来就写代码。**

## 适用场景

- **新增页面/窗口** — 完整全流程
- **新增 UI 组件** — 从设计开始（跳过头脑风暴）
- **UI 改版/重构** — 重新设计风格并决策
- **修复 UI BUG** — 从修复代码开始
- **仅为样式调整** — 确认方案后直接改代码

## 标准工作流

```
① 头脑风暴 → ② 设计风格 → ③ 设计决策 → ④ 用户确认
                                                 │
                                             通过/不通过
                                                 │
                                    不通过 ←──────┤
                                                 │ 通过
                                                 ▼
                                          ⑤ 实现代码 → ⑥ 验证 → ⑦ 交付
                                                 ↑
                                             修复 bug
```

## 各步骤说明

| 步骤 | 说明 | 调用资源 |
|------|------|---------|
| ① 头脑风暴 | 了解需求、探索方案 | brainstorming skill |
| ② 设计风格 | 配色、字体、布局 | ui-ux-pro-max skill / design-system skill |
| ③ 设计决策 | 技术选型（组件库/动画库） | ask 工具征求用户 |
| ④ 用户确认 | 展示设计稿让用户确认 | ask 工具 |
| ⑤ 实现代码 | 按设计编写组件和样式 | 前端实现 |
| ⑥ 验证 | 检查实现与设计一致 | 手动或视觉验证 |
| ⑦ 交付 | 提交代码 | git commit |

## 简化流程

| 场景 | 可跳过步骤 |
|------|-----------|
| 简单的样式调整 | ①②③ |
| 修复样式 bug | ①②③④ |
| 新增小功能按钮 | ①（直接从②开始） |
| 全新页面 | 全流程 |
