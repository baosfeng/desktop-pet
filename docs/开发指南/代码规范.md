# 代码规范

## 前端（TypeScript / React）

- 使用 TypeScript 严格模式
- 组件使用 PascalCase 命名
- 函数和变量使用 camelCase
- 常量使用 UPPER_CASE
- 使用 Function Components + Hooks
- Props 使用接口定义并导出
- 文件命名：组件使用 PascalCase（`Live2DView.tsx`），其他使用 camelCase
- 2 空格缩进

## 后端（Python / FastAPI）

- 遵循 PEP 8
- 类名使用 PascalCase
- 函数和变量使用 snake_case
- 常量使用 UPPER_CASE
- 类型注解必须完整
- 异步优先（`async def`）
- 4 空格缩进

## 通用

- 有意义的命名，禁止拼音缩写
- 删除注释掉的代码
- 敏感信息（API Key 等）使用环境变量或配置文件
