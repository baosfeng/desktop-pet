# 代码规范

## 通用规范

- **有意义的命名**，禁止拼音缩写
- **删除注释掉的代码**，不遗留死代码（dead code）
- **敏感信息**（API Key 等）使用环境变量或配置文件，不硬编码
- **提交前**必须通过对应语言的 linter 检查
- **注释用中文**，代码标识符用英文

## Go PetCore（正式代码）

> 遵循 [Effective Go](https://go.dev/doc/effective_go) + [Go Code Review Comments](https://go.dev/wiki/CodeReviewComments)
> Linter 配置：`petcore/.golangci.yml`

| 规则 | 要求 |
|------|------|
| 缩进 | tab（Go 标准） |
| 格式化 | [`gofumpt`](https://github.com/mvdan/gofumpt)（更严格的 `gofmt`） |
| 导入排序 | `golangci-lint` 的 `gci` 或 `importas`，分三组：标准库 / 第三方 / 内部 |
| 命名 | 驼峰，导出大写，未导出小写 |
| 错误处理 | 必须检查 `err`，禁止 `_ = fn()` 静默丢弃 |
| 接口 | 小接口优先（1-3 方法），大接口拆分 |
| 并发 | 使用 `sync` 包或 channel，避免共享内存 |
| CGO | `CGO_ENABLED=0`，禁止 cgo 依赖 |
| 包名 | 小写、单数、简短（无复数，无下划线） |
| 测试 | 表格驱动测试（table-driven tests），`t.Parallel()` 并行 |

### 推荐工具

```bash
# 安装 golangci-lint
brew install golangci-lint

# 运行检查
cd petcore
golangci-lint run ./...

# 安装 gofumpt
go install mvdan.cc/gofumpt@latest

# 格式化
gofumpt -l -w .
```

## Rust（Tauri 壳）

> 遵循 [Rust API Guidelines](https://rust-lang.github.io/api-guidelines/)
> Formatter：`rustfmt` | Linter：`clippy`

| 规则 | 要求 |
|------|------|
| 缩进 | 4 空格（rustfmt 默认） |
| 命名 | snake_case 变量/函数，PascalCase 类型，SCREAMING_SNAKE_CASE 常量 |
| 错误处理 | 使用 `anyhow` / `thiserror`，避免 `unwrap()` 和 `expect()` |
| 不安全代码 | 禁止 `unsafe`，除非有充分理由并加 `// SAFETY:` 注释 |
| 所有权 | 优先借用（`&T`），慎用 `clone()` |
| Tauri 命令 | 使用 `#[tauri::command]`，返回 `Result<T, E>` 而非 panic |
| 异步 | 使用 `async fn` + `tokio` |

### 推荐工具

```bash
# 格式化
cargo fmt

# 检查格式（CI 用）
cargo fmt --check

# lint 检查
cargo clippy -- -D warnings
```

## TypeScript / React（前端）

> 遵循 [TypeScript Strict Mode](https://www.typescriptlang.org/tsconfig#strict)
> Formatter：`Prettier` | Linter：`ESLint` v9 flat config

| 规则 | 要求 |
|------|------|
| 缩进 | 2 空格 |
| 命名 | 组件 PascalCase（`ChatBubble.tsx`），其他 camelCase |
| 类型 | 严格模式，禁止 `any`，优先 `interface` 后 `type` |
| 组件 | Function Components + Hooks，禁止 class 组件 |
| Props | 使用 `interface` 定义并 `export` |
| 状态管理 | 使用 `useState` / `useReducer`，避免滥用外部状态库 |
| 副作用 | `useEffect` 必须有依赖数组，避免无限循环 |
| IPC 通信 | 通过 `bridge.ts` 封装，不在组件中直接调用 Tauri API |
| 导入顺序 | `react` → 第三方 → 内部组件 → 样式 |

### 推荐工具

```bash
# lint 检查
cd frontend
pnpm lint

# 格式化
pnpm format

# 格式化检查（CI 用）
pnpm format:check
```

## 配置文件位置

| 语言 | 配置文件 | 作用 |
|------|---------|------|
| Go | `petcore/.golangci.yml` | golangci-lint 规则 |
| Rust | `src-tauri/rustfmt.toml` | rustfmt 风格 |
| Rust | `src-tauri/clippy.toml` | clippy 额外规则 |
| TypeScript | `frontend/eslint.config.js` | ESLint flat config |
| TypeScript | `frontend/.prettierrc` | Prettier 格式化 |

## Pre-commit Hook

项目使用 `lefthook` 管理 pre-commit 钩子，配置位于 `lefthook.yml`：

```bash
# 安装 lefthook
brew install lefthook

# 激活钩子
lefthook install
```

钩子会在每次 `git commit` 前自动运行：
- 检查代码格式
- 运行 linter
- 运行相关测试
