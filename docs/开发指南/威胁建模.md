# 威胁建模 (STRIDE)

> 使用 STRIDE 方法论对 Desktop Pet 进行威胁建模。
> 桌面端应用威胁面较小，重点保护 API Key 和用户本地数据。

---

## 数据流图

```
┌──────────┐    Tauri IPC     ┌──────────┐   stdin/stdout  ┌────────────┐
│  WebView  │ ←────────────→  │ Rust 壳  │ ←────────────→ │ Go PetCore  │
│ (前端)    │                  │ (Tauri)  │                 │  (内核)      │
└──────────┘                  └──────────┘                 └──────┬─────┘
                                                                  │
                                                           ┌──────▼─────┐
                                                           │  LLM API    │
                                                           │ (外部 HTTP) │
                                                           └────────────┘
```

## STRIDE 分析

### S — 身份冒充 (Spoofing)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| 攻击者伪造 sidecar 命令注入 | 中 | stdin/stdout 是 Tauri 管理的私有管道，外部进程无法写入 |
| 攻击者替换 PetCore 二进制 | 高 | 代码签名 + Tauri bundle 完整性校验 |

### T — 篡改 (Tampering)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| 攻击者修改本地配置文件 | 低 | 配置不含敏感信息（API Key 在环境变量中） |
| 攻击者修改记忆数据库 | 中 | 数据仅本地存储，无远程同步。可考虑文件完整性校验（Phase 3+） |
| 攻击者注入恶意 YAML 动作包 | 中 | L1 插件沙箱执行（Phase 4 实现） |

### R — 抵赖 (Repudiation)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| 用户否认执行了桌面控制操作 | 低 | Phase 6 实现审计日志（不可篡改） |

### I — 信息泄露 (Information Disclosure)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| **API Key 泄露** | 🔴 **高** | 通过环境变量读取（`LLMConfig.APIKey()`），不硬编码、不落盘 |
| 对话记录泄露 | 中 | 存储在 `~/.desktop-pet/`，默认权限 600 |
| LLM 请求被中间人攻击 | 高 | 强制 HTTPS（由 Provider 实现保证） |

### D — 拒绝服务 (Denial of Service)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| LLM API 返回超大数据 | 低 | Agent Pipeline 有 MaxTokens 限制 |
| 恶意插件耗尽 CPU | 中 | L2 JS 沙箱限制执行时间（Phase 4 实现） |
| sidecar 子进程崩溃 | 中 | Tauri sidecar 自动重启（最多 3 次） |

### E — 权限提升 (Elevation of Privilege)

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| L2 JS 脚本逃逸沙箱 | 高 | goja 引擎天然隔离，无文件/网络访问权限 |
| MCP 子进程越权 | 中 | 独立进程 + 权限声明（Phase 4 实现） |
| 桌面控制操作被滥用 | 高 | 每次操作需用户确认 + 可撤销授权（Phase 6） |

---

## 关键风险应对优先级

| 优先级 | 风险 | 措施 | 截止 |
|--------|------|------|------|
| P0 | API Key 泄露 | 环境变量读取 + 90 天轮换策略 | ✅ 已实现 |
| P1 | HTTPS 强制 | Provider 实现保证 | Phase 1 |
| P2 | L2 JS 沙箱 | goja + 时间/内存限制 | Phase 4 |
| P3 | 审计日志 | 桌面控制操作日志 | Phase 6 |
