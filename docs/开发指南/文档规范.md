# 文档规范

> ⚠️ **何时阅读：** 当你需要创建、修改或删除项目文档时，必须先阅读本文档，了解文档结构、命名规则和渐进式加载约定。

## 语言要求
所有项目文档必须使用中文编写。代码、标识符、文件路径、shell 命令保持原文。

## 目录与命名
- 所有文档统一放在 `docs/` 目录下
- 文档文件和目录名必须使用中文
- `AGENTS.md` 例外：位于项目根目录

## AGENTS.md 规范
AGENTS.md 必须包含：项目标识、项目简介、文档导航、功能模块、开发规范。

## 隔离文档禁令
每篇文档必须至少被索引或其他文档引用一次。新增文档必须同步更新索引。

## 计划和任务类临时文档清理
任务完成后立即删除，不得纳入正式文档体系。

## 代码示例规范
遵循语言编码规范、使用最新语法。

## ✅ 操作前自查清单
- [ ] 文档文件名使用了中文
- [ ] 新增文档已在 docs/索引.md 或相关文档中添加引用链接
- [ ] 文档内容不构成孤立文档（至少有一条入链）
- [ ] 遇到报错时已先查看 `docs/踩坑/` 对应功能域是否有类似记录
