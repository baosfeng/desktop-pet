## 一、代码质量问题

### 1.1 生产环境使用 Mock 实现（🔴 高）

- **位置：** `petcore/cmd/petcore/main.go` L104、L107
- **问题：** `fsm.NewMockMachine` 和 `memory.NewInMemoryManager` 被当作正式引擎使用
- **影响：** 重启后所有记忆丢失，`go.mod` 中连 SQLite driver 都没有引入
- **架构规划要求：** Phase 1 应实现 SQLite 持久化（`docs/参考/架构规划方案.md` L593 "SQLite + FTS5"）

### 1.2 Rust 代码零测试覆盖（🔴 高）

- **位置：** `src-tauri/src/` 目录下 5 个 `.rs` 文件
- **问题：** `commands.rs`、`sidecar.rs`、`tray.rs`、`window.rs`、`main.rs` 全部没有单元测试
- **质量基线要求：** `docs/开发指南/质量基线.md` 要求覆盖率 ≥60%，Rust 测试必须在 CI 通过

### 1.3 手写字符串搜索重复造轮子（🟡 中）

- **位置：** `petcore/internal/memory/manager.go` L150-156
- **问题：** `containsStr()` 手写了子串匹配算法
- **建议：** 直接使用标准库 `strings.Contains(s, substr)`

### 1.4 多个占位/脚手架测试残留（🟡 中）

- **位置：** `frontend/src/` 下 7 个测试文件
  - `minimal.test.tsx`
  - `alias-test.test.tsx`
  - `import-test.test.tsx`
  - `motion-test.test.tsx`
  - `motion-render-test.test.tsx`
  - `render-test.test.tsx`
  - `zustand-test.test.tsx`
  - `hoisted-test.test.tsx`
- **问题：** 仅验证「能 render / 能 import」，无业务断言，疑似调试脚手架残留
- **影响：** 虚假抬高覆盖率数字，掩盖真实测试缺失

### 1.5 CLI 模式完全未实现（🟡 中）

- **位置：** `petcore/cmd/petcore/main.go` L154-158
- **代码：** `runCLI()` 仅打印 "not yet implemented" 后阻塞等待
- **架构规划要求：** `docs/参考/架构规划方案.md` L222-236 定义了完整的 CLI Shell 子命令设计

### 1.6 Engine.Run 主循环是骨架（🔴 高）

- **位置：** `petcore/internal/core/engine.go` L79
- **问题：** 核心事件循环只有 `<-ctx.Done()`，标注 `TODO: Phase 2`
- **影响：** Engine 启动后不做任何实际工作，Server 和 Engine 缺乏协调

### 1.7 Live2D 表情/动作联动未接线（🟡 中）

- **位置：** `frontend/src/lib/live2d.ts` L110-140
- **问题：** `setModelExpression()` / `setModelMotion()` 函数已实现，但在 `App.tsx` 和所有 `hooks/` 中从未被调用
- **影响：** 宠物情绪变化仅靠文字状态栏表示（"晒太阳中~"），无动画反馈

### 1.8 mock → openai 的不可达 fallback（🟢 低）

- **位置：** `petcore/internal/core/engine.go` L149-151
- **问题：** `UpdateLLMConfig` 中 `providerName == "mock"` 改为 `"openai"`，但此时 `providerName` 来源于前端设置面板的下拉选择，mock 不在可选列表中

## 三、UI 设计问题

### 3.1 窗口透明度 slider 无后端联动（🟡 中）

- **位置：** `frontend/src/components/SettingsPanel.tsx` L283-295
- **问题：** opacity slider 只存储在 Zustand store → localStorage，没有对应的 Tauri 命令来设置 WebView 透明度
- **bridge.ts** 中没有 `setWindowOpacity` 命令

### 3.2 Base URL 输入框只读（🟢 低）

- **位置：** `frontend/src/components/SettingsPanel.tsx` L258-268
- **问题：** 接口地址从预设 Provider 自动填充且 `readOnly`，自定义 API 地址的用户无法输入
- **建议：** 增加 "自定义" Provider 选项或允许覆盖 baseUrl

### 3.3 Live2D 模型硬编码（🟡 中）

- **位置：** `frontend/src/App.tsx` L34
- **代码：** `const modelPath = "https://cdn.jsdelivr.net/..."`
- **影响：** 永远使用 Haru 模型，无本地模型目录扫描、无模型选择 UI

### 3.4 无暗黑模式切换（🟢 低）

- **位置：** `frontend/src/app.css` L162-194 定义了 `.dark` 主题
- **问题：** daisyUI 配置 `prefersdark: false` 硬关闭，无手动切换入口

### 3.5 无国际化 i18n（🟢 低）

- **影响范围：** 全部 UI 字符串硬编码为中文
- **质量基线要求：** Phase 3+ 须通过 `react-intl` / `i18next` 外部化

### 3.6 聊天消息无持久化（🟡 中）

- **位置：** `frontend/src/stores/petStore.ts`
- **问题：** `messages` 仅存内存 Zustand store，刷新即丢失

### 3.7 宠物待机无实际动画（🟡 中）

- **位置：** `frontend/src/App.tsx` L87-89
- **问题：** 状态栏纯文字显示（"晒太阳中~"），没有 idling 动画逻辑调度随机眨眼/晃头

### 3.8 打字机光标与逐字显示不同步（🟢 低）

- **位置：** `frontend/src/components/ChatBubble.tsx` + `frontend/src/app.css`
- **问题：** 光标使用纯 CSS `animate-blink`（独立 0.6s 循环），而不是 JS 驱动的与 `visibleChars` 同步的闪烁

## 五、安全性问题

### 5.1 API Key 明文存储在 localStorage（🔴 高）

- **位置：** `frontend/src/stores/petStore.ts` L109
- **代码：** `localStorage.setItem(STORAGE_KEY, JSON.stringify(settings))`
- **风险：** XSS 攻击可直接读取 localStorage，Settings 对象包含完整 `apiKey`
- **建议：** 使用系统 keychain（macOS Keychain / Windows Credential Manager）或 Tauri 加密存储

### 5.2 API Key 通过多重明文通道传输（🔴 高）

- **传输链路：**
  1. `SettingsPanel.tsx` → `updateConfig()` → `invoke("update_config", {config})` — Tauri IPC 明文 JSON
  2. `commands.rs` → `sidecar.rs` → PetCore stdin — sidecar 进程间明文 JSON
- **风险：** 任何能截获 IPC 通信或进程间通信的攻击者可获取 API Key

### 5.3 API Key 验证从前端直接发起 HTTP 请求（🟡 中）

- **位置：** `frontend/src/components/SettingsPanel.tsx` L26-58
- **问题：** `verifyApiKey()` 在前端直接 `fetch(baseUrl + "/chat/completions")`，暴露 API Key 在浏览器网络请求中，且可能受 CORS 限制
- **建议：** 改为通过 Tauri IPC → PetCore 后端统一调用 LLM API
