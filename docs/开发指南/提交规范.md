# 提交规范

> ⚠️ **何时阅读：** 当你需要提交代码到 Git 仓库时，必须先阅读本文档，了解提交信息格式和提交策略。

## 提交信息格式

`<type>: <中文描述>`

### type 列表
| type | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档变更 |
| style | 代码格式（不影响功能） |
| refactor | 代码重构 |
| test | 测试相关 |

## 提交策略
- 按功能单次提交
- 提交前确保所有测试通过
- 一次性提交完整的代码变更

## ✅ 操作前自查清单
- [ ] 提交信息格式正确（`type: 中文描述`）
- [ ] 所有测试已通过
- [ ] 提交范围合理（一个功能一个提交）
