# 构建与测试

> **提交前必须运行全部测试，修复所有失败的测试用例，确保全部通过。**

## 运行环境要求

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Rust | 1.80+ | Rust 工具链（含 `cargo`、`rustfmt`、`clippy`） |
| Go | 1.23+ | Go 语言 |
| Node.js | 22+ | 推荐 22+ LTS |
| pnpm | 11+ | 包管理器 |
| Tauri CLI | v2 | `cargo install tauri-cli --version "^2"` |
| lefthook | — | Git hooks 管理，`go install github.com/evilmartians/lefthook@latest` 或通过 pnpm 安装 |

## 测试命令

### Go PetCore 测试

```bash
# 运行所有测试
cd petcore
go test ./...

# 带竞态检测
go test -race ./...

# 查看覆盖率
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out
```

### 前端测试

```bash
# 运行所有测试
cd frontend
pnpm test

# 带覆盖率
pnpm test -- --coverage
```

### Rust 壳测试

```bash
# 运行 Rust 测试
cd src-tauri
cargo test

# 运行所有测试（含集成测试）
cargo test --all-targets
```

### 全部测试一键运行

```bash
# 所有语言测试
cd petcore && go test ./... && cd ../frontend && pnpm test && cd ../src-tauri && cargo test
```

## Lint 检查

### Go

```bash
cd petcore
golangci-lint run ./...     # 综合检查（govet + staticcheck + gofumpt 等）
```

### Rust

```bash
cd src-tauri
cargo fmt --check            # 格式检查
cargo clippy -- -D warnings  # lint 检查（deny 模式）
```

### TypeScript

```bash
cd frontend
pnpm lint                   # ESLint 检查
pnpm format:check           # Prettier 格式检查
```

## 构建命令

### Go PetCore

```bash
cd petcore
CGO_ENABLED=0 go build -ldflags="-s -w" -o ../src-tauri/binaries/petcore ./cmd/petcore/
```

> `-ldflags="-s -w"` 去掉调试信息，减小二进制体积。

### 前端

```bash
cd frontend
pnpm build
```

### Rust 壳

```bash
cd src-tauri
cargo build --release
```

### 完整打包

```bash
cd src-tauri
cargo tauri build

# 产物：src-tauri/target/release/bundle/dmg/DesktopPet-1.0.0.dmg
```

## CI/CD

GitHub Actions 配置位于 `.github/workflows/ci.yml`，每次推送自动执行：

1. **Lint** — 检查各语言代码规范（Go/Rust/Frontend 并行）
2. **Test** — 运行全部测试（含竞态检测）
3. **Coverage** — Go 覆盖率检查（≥60%）
4. **Build** — 验证打包是否成功

## Pre-push 钩子（推荐）

```bash
# 安装 Git hooks（一次性）
make setup-hooks
```

安装后，`git push` 会自动运行与 CI 完全相同的 lint 命令：
- `golangci-lint run ./...`（Go）
- `cargo clippy -- -D warnings`（Rust，含 sidecar 构建）
- `pnpm lint --no-cache`（前端）
- `go build`（验证 Go 编译）

```bash
# 手动运行 pre-push 检查
make pre-push
```

所有检查通过后才允许推送到远程，避免浪费 CI 分钟额度。

也可通过 `scripts/dev.sh` 启动开发环境，自动安装 hooks。
