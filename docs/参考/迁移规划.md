## 1. 迁移目标

| 指标 | 当前 (Electron + Python) | 目标 (Tauri + Go) | 改善 |
|------|------------------------|-------------------|------|
| **安装包体积** | ~210-260 MB | **~15-25 MB** | **~10x** |
| **运行时内存** | ~100-200 MB | **~30-60 MB** | **~3-5x** |
| **启动速度** | ~3-5 秒 | **~1-2 秒** | **~2-3x** |
| **鼠标穿透** | ✅ Electron 内置 | ✅ Tauri 原生 | 持平 |
| **自动更新** | ✅ electron-updater | ✅ tauri-plugin-updater | 持平 |
| **代码复用** | 前后端分离，单 Shell | **多 Shell 复用 PetCore** | **质变** |
| **开发语言** | Python + TS | **Go + Rust + TS** | 学习成本略增 |

### 1.1 "一步到位"的含义

本次迁移是一次性将架构从 Electron + Python 切换到 Tauri v2 + Go，而非渐进式迁移。迁移后：

- **不再有 Electron** — 被 Tauri v2 完全替代
- **不再有 Python** — 所有后端逻辑用 Go 重写
- **前端基本保留** — React + TypeScript + PixiJS 不变，仅通信方式从 WebSocket 改为 Tauri IPC
- **状态机 + AI + 记忆 + 插件** — 全部在 Go PetCore 中实现

### 通信机制对比

| 层面 | 旧架构 | 新架构 |
|------|-------|-------|
| **壳 ←→ 前端** | Electron IPC (主进程 ↔ 渲染进程) | **Tauri IPC** (Rust `#[tauri::command]` ↔ JS `invoke()`) |
| **壳 ←→ 后端** | WebSocket (127.0.0.1:PORT) | **sidecar stdin/stdout** 或 **本地 socket** |
| **后端 → 前端** | WebSocket JSON push | **Tauri event** (`app.emit()` → JS `listen()`) |
| **前端命令** | HTTP POST / WebSocket send | **Tauri invoke** (`invoke('command_name', args)`) |

## 6. 关键设计决策

### 6.1 Tauri ↔ Go 通信方式

| 方式 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| **stdin/stdout JSON** | 零依赖、跨平台、Tauri sidecar 原生支持 | 双工需自行管理 | ✅ **首选** |
| **本地 TCP socket** | 双向流、可独立调试 | 端口冲突、安全需额外处理 | ❌ |
| **Unix domain socket** | 安全、高性能 | macOS/Linux only | ⚠️ 备选 |
| **Tauri shell plugin** | 直接管理子进程 | 无标准 JSON 协议 | ⚠️ 基础 |

**选型理由**：stdin/stdout JSON 是 Tauri sidecar 的原生模式，零额外依赖，Tauri 负责子进程生命周期管理。

### 6.2 记忆存储方案

| 阶段 | 方案 | 理由 |
|------|------|------|
| **Phase 1** | SQLite + FTS5 | Go 内置 (`modernc.org/sqlite`)，零 CGO，支持全文检索 |
| **Phase 2+** | + 嵌入式向量库 | 需要语义搜索时接入（如 `weaviate` 或自建简单向量索引） |

### 6.3 LLM 适配方案

参考 Reasonix 的 Provider 接口设计，使用 blank import 自注册：

```go
// main.go
import (
    _ "github.com/baosfeng/desktop-pet/petcore/internal/llm/openai"
    _ "github.com/baosfeng/desktop-pet/petcore/internal/llm/ollama"
)
```

每个 Provider 在 `init()` 中调用 `llm.Register("openai", factory)`。

### 6.4 前端保持不变的组件

以下组件只需修改通信层，UI 逻辑保留：

- ✅ PetCanvas (PixiJS + Live2D) — 修改数据获取方式
- ✅ ChatBubble — 修改发送/接收为 bridge.ts
- ✅ SettingsPanel — 修改配置读写为 bridge.ts
- ❌ WebSocket 客户端 — 整体替换为 bridge.ts

## 7. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| **Rust 学习曲线** | 开发进度慢 | Tauri 壳代码量小 (~500 行 Rust)，主要逻辑在 Go 中 |
| **Go 向量库不成熟** | L3 记忆精度不足 | Phase 1 先用 FTS5，Phase 2 评估嵌入式向量方案 |
| **JS 脚本引擎性能** | L2 脚本插件慢 | GoJA (goja) 性能足够，且可通过沙箱限制复杂度 |
| **sidecar 通信稳定性** | 子进程崩溃 | Tauri sidecar 自动重启 + 心跳检测 |
| **Live2D WebGL 兼容性** | 部分系统不显示 | Tauri WebView (WKWebView/WebView2) 均支持 WebGL |
| **macOS 未签名警告** | 用户首次打开受阻 | 文档说明 + 计划获取 Developer ID 签名 |
| **Python 代码无法复用** | 后端需完全重写 | Go 版代码量约 3000-5000 行，可逐步实现 |
