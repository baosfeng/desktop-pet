## 2. 桌面端框架与架构参考

### 2.1 DeepSeek-Reasonix (Reasonix) ⭐26,600 — **Go 内核 + 多壳架构**

| 字段 | 内容 |
|------|------|
| **仓库** | https://github.com/esengine/DeepSeek-Reasonix |
| **语言** | Go (主) + TypeScript (前端) |
| **Star** | ⭐26,600 |
| **架构** | Go 内核 + Wails 桌面壳 / bubbletea TUI |
| **许可** | MIT |

**架构亮点：**

```
┌──────────────────────────────────────────┐
│          Go 内核 (CGO_ENABLED=0)          │
│   internal/control.Controller            │
│   ─ providers / agent / memory / tool    │
│   ─ plugin / store / sandbox / event     │
└──────────┬──────────────────┬────────────┘
           │                  │
           ▼                  ▼
┌──────────────────┐  ┌────────────────────┐
│  CLI (bubbletea) │  │ Desktop (Wails)    │
│  TUI 终端界面     │  │ WebView + React    │
│  单静态二进制      │  │ ~20MB 二进制        │
│  CGO_ENABLED=0   │  │ 需要 CGO (WebKit)  │
└──────────────────┘  └────────────────────┘
```

**关键设计模式：**

| 设计 | 描述 | 可学习之处 |
|------|------|-----------|
| **嵌套 Module** | `desktop/` 是独立 Go module，`replace reasonix => ..` 引用内核 | CLI 保持纯静态，桌面端可引入 CGO |
| **内核复用** | CLI 和 Desktop 共享同一套 `control.Controller` | 一份代码，多壳复用 |
| **Wails 绑定** | Go 方法通过 `Bind` 直接暴露给 JS，事件通过 `runtime.EventsEmit` 推送 | 无需 HTTP，零延迟通信 |
| **跨平台适配** | macOS TitleBarHiddenInset、Windows WebView2 策略、Linux GPU 策略 | 平台差异处理范例 |
| **编辑器 seam** | CodeViewer/DiffView 通过懒加载 + 统一 Props 接口切换 | 可替换架构设计 |

**参考价值：** 业界最成熟的 Go + WebView 桌面应用架构。嵌套 module 设计、内核复用模式、Wails 最佳实践。

## 4. Live2D 渲染相关

### 4.1 渲染方案对比

| 方案 | 适用框架 | 说明 |
|------|---------|------|
| **pixi-live2d-display** | PixiJS 8 | 最成熟的 Web Live2D 方案，所有桌宠项目都在用 |
| **pixi-live2d-display-lipsyncpatch** | PixiJS 8 | 带口型同步的 fork，用于语音交互 |
| **Cubism Native SDK** | C++ | 原生性能最佳，但集成复杂度高 |
| **cubism-go** | Go + Ebitengine | 非官方 Go 绑定（33⭐），不成熟 |

### 4.2 推荐渲染架构

```
Electron / Tauri / Wails (WebView)
         │
         ▼
   PixiJS 8 + pixi-live2d-display
         │
         ▼
   Live2D Cubism 4 模型 (.model3.json)
         │
         ▼
   WebGL 渲染 (WebView 内)
```

无论前端用什么框架（Electron / Tauri / Wails），Live2D 渲染都走 WebView 内的 PixiJS 方案——这是所有现有项目的统一做法。

## 6. 总结与推荐路线

### 分阶段参考建议

| 阶段 | 参考项目 | 关注点 |
|------|---------|--------|
| **窗口管理** | bongo-cat-next (Tauri) / Reasonix (Wails) | 透明窗口、鼠标穿透、置顶、系统托盘 |
| **Live2D 渲染** | bongo-cat-next / git2968/Desktop-pet | pixi-live2d-display 集成、表情动画 |
| **AI 后端** | my-neuro / Reasonix | LLM 适配、Agent Pipeline、记忆系统 |
| **MCP 工具** | git2968/Desktop-pet / Reasonix | 工具注册、MCP 客户端协议 |
| **语音交互** | Open-LLM-VTuber | VAD→ASR→LLM→TTS→口型同步 |
| **自动更新** | bongo-cat-next (Tauri) / Reasonix (Wails) | updater 集成、增量更新 |

### 核心理念提炼

从所有参考项目中提炼的关键设计原则：

1. **内核与壳分离**：Reasonix 的嵌套 module 设计，Go 内核可在 CLI / Desktop / Server 间复用
2. **Live2D 统一方案**：无论什么桌面框架，都用 WebView 内 PixiJS 渲染——这是唯一成熟路径
3. **AI 适配抽象层**：所有成功项目都做了 Provider 抽象，不绑定单一大模型
4. **MCP 成为标准**：多个项目都在用 MCP 协议做工具扩展，是行业趋势
5. **记忆系统分层**：短期(滑动窗口) + 核心(SQLite) + 长期(向量)是经过验证的设计

## 15. 核心设计模式提炼

### 按相关性排序

| 模式 | 代表项目 | 在桌宠中的应用 |
|------|---------|--------------|
| **透明覆盖窗口** | Flameshot, Übersicht | 宠物渲染在任何窗口上方 |
| **窗口锁定到桌面** | Lively, Rainmeter, Conky | 宠物停留在图标/壁纸后面 |
| **系统托盘常驻 + 弹出** | Maccy, Bob, CopyQ | 宠物后台运行，单击唤醒 |
| **WebView + 原生后端** | Tauri, Wails, Übersicht | 宠物 = 原生引擎 + Web 渲染 (Live2D/Pixi) |
| **热重载包/插件** | Espanso, CopyQ | 动作/动画在运行时热重载 |
| **核心-UI 分离** | Glances, Reasonix | `PetCore` 支持 CLI/GUI/WebSocket |
| **MCP 协议** | 5ire, CodePilot, Witsy | 宠物作为 AI 代理，可扩展工具 |
| **即时模式 GUI** | egui | 轻量级宠物 GPU 渲染 |
| **Elm 架构** | Iced | 宠物 UI 的 MVU 状态管理 |
| **桌面壁纸注入** | Lively Wallpaper | 宠物渲染到桌面图标层 |

### 最终推荐技术栈组合

```
                    ┌──────────────────────────┐
                    │   PetCore (Go/Rust)       │
                    │   - 状态引擎               │
                    │   - AI Agent               │
                    │   - 记忆系统               │
                    │   - MCP 工具               │
                    └────────┬────────┬─────────┘
                             │        │
              IPC (stdin/WS  │        │ Wails/Tauri bindings
                             │        │
              ┌──────────────▼─┐  ┌──▼──────────────┐
              │ CLI (bubbletea)│  │ Desktop (Wails) │
              │ 终端调试/管理   │  │ WebView + React  │
              │               │  │ + PixiJS + Live2D│
              └────────────────┘  └─────────────────┘
```
