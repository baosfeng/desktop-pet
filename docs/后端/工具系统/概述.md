# 工具系统

## 职责

提供 AI Agent 可调用的工具集合。工具通过统一注册中心管理，采用 Tool 接口 + Registry 模式。Phase 1 内置 `remember` 工具，Phase 4 引入 MCP 协议扩展。

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/tool/registry.go` | `Tool` | 工具接口（Name/Description/Execute） |
| `petcore/internal/tool/registry.go` | `Registry` | 工具注册中心（Register/Execute/List） |
| `petcore/internal/tool/registry.go` | `MockTool` | Mock 实现，供测试使用 |
| `petcore/internal/tool/builtin/remember.go` | `RememberTool` | 内置 remember 工具，保存用户事实到记忆 |

## Tool 接口

```go
type Tool interface {
    Name() string
    Description() string
    Execute(ctx context.Context, args string) (string, error)
}
```

## 工具调用循环

Agent 在执行对话时自动管理工具调用（Phase 4 实现工具循环）：

```
LLM 回复 → 解析 Tool Call
               │
               ▼
        Registry.Execute(name, args)
               │
               ▼
        执行内置工具 / MCP 工具
               │
               ▼
        结果注入 → 继续 LLM 调用
               │
               ▼
        (重复直到无工具调用)
```

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 记住、保存事实 | `RememberTool.Execute()` | 内置 remember 工具 |
| 工具注册 | `Registry.Register()` | 注册新工具 |
| MCP、工具扩展 | (Phase 4) | 外部 MCP 工具服务器 |
| 搜索、联网 | (Phase 4) | 内置 web_search 工具 |
