# 状态机

## 职责

管理宠物的行为状态转换。采用四状态 FSM 模型，每个状态对应不同的窗口行为和动画模式。

## 状态定义

```
                    ┌──────────┐
          ┌────────►│  待机     │◄──────────────┐
          │         │ (穿透/   │                │
          │         │  随机动作)│                │
          │         └────┬─────┘                │
          │              │                      │
      对话结束         检测到用户               超时
          │              │                      │
          │         ┌────▼─────┐                │
          ├────────►│  关注     │────────────────┤
          │         │ (悬停/   │                │
          │         │  看向鼠标)│                │
          │         └────┬─────┘                │
          │              │                      │
          │         用户对话/点击                 │
          │              │                      │
          │         ┌────▼─────┐                │
          │         │  交互     │────思考中──────┤
          │         │ (对话/   │ (等待 LLM 回复) │
          │         │  表情动画)│                │
          │         └────┬─────┘                │
          │              │                      │
          │         LLM 流式回复                  │
          │              │                      │
          │         ┌────▼─────┐                │
          │         │  说话     │                │
          │         │ (打字机  │────────────────┤
          │         │  效果)   │                │
          │         └──────────┘                │
          └─────────────────────────────────────┘
```

## 每个状态对应的窗口行为

| 状态 | 鼠标穿透 | 置顶 | 动画 |
|------|---------|------|------|
| 待机 | ✅ 开启 | ✅ | 随机小动作（眨眼、晃头） |
| 关注 | ❌ 关闭（可交互） | ✅ | 转向鼠标方向 |
| 交互 | ❌ 关闭 | ✅ | 表情随对话变化 |
| 说话 | ❌ 关闭 | ✅ | 口型同步 |

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/fsm/machine.go` | `Machine` | FSM 引擎（状态定义/转换规则/回调） |
| `petcore/internal/fsm/machine.go` | `MockMachine` | 内存实现，持真实转换和 OnTransition 回调 |
