# AI Agent 引擎

## 职责

核心对话引擎（位于 Go PetCore 的 `internal/agent/`），负责管理用户与宠物之间的对话流程，包括消息路由、上下文构建、LLM 调用和工具调度。通过 Pipeline 洋葱模型实现职责分离。

## 核心功能

- **对话管理**：接收用户消息，构建上下文，调用 LLM 生成回复
- **上下文构建**：将系统人设 + 记忆（L1/L2/L3）+ 工具定义组合为 LLM 输入
- **Pipeline 管线**：洋葱模型 Stage 链，每个 Stage 负责一个正交职责
- **角色人设**：管理宠物的性格、说话风格、角色设定
- **工具调用循环**：LLM 请求 → 工具执行 → 结果注入 → 继续请求（最多 N 轮）

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/agent/agent.go` | `Agent` | Agent 接口 + agentImpl.Run，对话管理 |
| `petcore/internal/agent/pipeline.go` | `Pipeline` / `Stage` | Pipeline 洋葱模型，4 个内置 Stage |
| `petcore/internal/agent/pipeline.go` | `pipelineCtx` | 管线上下文，传递 Request/SystemPrompt/Sink |
| `petcore/internal/core/engine.go` | `Engine` | PetCore 主引擎，DI 组装所有模块 |

## 对话流程

```
```
用户消息 → Engine.HandleInput()
                           │
                           ▼
           Pipeline.Process() — 洋葱模型
               │
               ├── [PreProcess Stage] 消息标准化/过滤 (骨架, Phase 2 实现)
               ├── [Memory Stage] 注入记忆到上下文 (骨架, Phase 2 实现)
               ├── [LLMCall Stage] 调用 Provider.Stream()
               │     ├── Text Chunk → agent.reply 事件 (打字机效果)
               │     ├── Tool Call → (骨架, Phase 4 实现)
               │     └── Done → 完成
               └── [PostProcess Stage] 提取新事实 → 写入记忆 (骨架, Phase 2 实现)
                           │
                           ▼
           事件推送 → Sink → Tauri event → 前端 UI 更新
```
```

## Pipeline 管线设计

```
洋葱模型（参考 NyaDeskPet + Reasonix）：
  用户输入 → [PreProcess] → [Memory] → [LLMCall] → [PostProcess] → 输出
                                              ↓(递归, Phase 4)
                                       工具执行结果注入
```

每个 Stage 实现接口：
```go
type Stage interface {
    Name() string
    Process(ctx context.Context, pCtx *pipelineCtx, next StageFunc) error
}

type StageFunc func(ctx context.Context, pCtx *pipelineCtx) error
```

## 业务场景映射

| 需求关键词 | 入口类 | 入口方法 |
|-----------|--------|---------|
| 对话、聊天、说话 | `Agent` | `Run()` |
| Pipeline、管线、Stage | `Pipeline` | `Process()` |
| LLM 流式调用 | `LLMCallStage` | `Process()` |
| 工具调用、Function Call | `Agent` | `Run()` (Phase 4 实现工具循环) |
