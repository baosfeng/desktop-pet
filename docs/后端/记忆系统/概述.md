# 记忆系统

## 职责

管理宠物的所有记忆，实现「越用越懂你」的渐进式学习。采用三层记忆架构（L1 短期 / L2 核心 / L3 长期）。Phase 1 使用纯内存实现（`InMemoryManager`），Phase 2 切换为 SQLite 持久化。

## 记忆分层

| 层级 | 存储 | 生命周期 | 内容 |
|------|------|---------|------|
| L1 短期 | 内存（切片，NewestFirst） | 当前会话 | 最近 N 轮对话、当前上下文 |
| L2 核心 | 内存 map（Phase 2 → SQLite KV） | 永久 | 用户姓名、偏好、结构化事实 |
| L3 长期 | 内存 map（Phase 2 → SQLite + FTS5） | 长期 | 对话摘要、历史事件 |

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/memory/manager.go` | `Manager` | 记忆管理器接口，统一读写入口 |
| `petcore/internal/memory/manager.go` | `InMemoryManager` | 全内存实现（3 层），sync.RWMutex 并发安全 |

## Manager 接口

```go
type Manager interface {
    // L1 短期记忆
    AddShortTerm(msg Message)
    GetShortTerm() []Message
    ClearShortTerm()

    // L2 核心记忆
    Remember(key, value string, importance int) error
    Recall(key string) (string, error)
    GetAllCore() (map[string]string, error)

    // L3 长期记忆
    Store(ctx context.Context, fact Fact) error
    Search(ctx context.Context, query string, limit int) ([]Fact, error)
}
```

## 记忆工作流

```
用户说话
   │
   ▼
L1 短期 ← 直接追加到上下文
   │
   ├── AI 在回复中发现新信息
   │      │
   │      ├── 结构化信息（姓名/偏好）→ L2 核心 (Remember)
   │      │
   │      └── 事件性信息 → L3 长期 (Store)
   │
   └── 下次对话
          │
          ▼
      1. 读取 L2 核心 → 用户偏好
      2. 检索 L3 长期 → Search
      3. 融合 → 构建 context → LLM 推理
```

## 事实提取策略

两种方式互补：
1. **LLM 主动提取（推荐的）**：LLM 通过 `remember` 工具写入新发现的事实
2. **后台异步提取（Phase 2）**：对话结束后 Extractor 分析对话内容提取新事实

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 记住、记得 | `Manager.Remember()` | 写入 L2 核心事实 |
| 查询、回忆 | `Manager.Recall()` | 检索 L2 核心记忆 |
| 搜索记忆 | `Manager.Search()` | FTS5 检索 L3 长期（Phase 2） |
| 短期对话 | `Manager.GetShortTerm()` | 获取最近对话记录 |
