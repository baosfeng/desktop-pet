# 记忆系统

## 职责

管理宠物的所有记忆，实现「越用越懂你」的渐进式学习。采用三层记忆架构，确保隐私和效率。存储于 `~/.desktop-pet/memory.db`（SQLite + FTS5）。

## 记忆分层

| 层级 | 存储 | 生命周期 | 内容 |
|------|------|---------|------|
| L1 短期 | 内存（滑动窗口） | 当前会话 | 最近 20 轮对话、当前上下文 |
| L2 核心 | SQLite（结构化 KV） | 永久 | 用户姓名、生日、偏好、重要关系 |
| L3 长期 | SQLite + FTS5（逐步引入向量检索） | 长期 | 历史对话摘要、有趣事件、学到的习惯 |

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/memory/manager.go` | `MemoryManager` | 记忆管理器，统一读写入口 |
| `petcore/internal/memory/short_term.go` | `ShortTerm` | 短期记忆（滑动窗口，环形缓冲区） |
| `petcore/internal/memory/core.go` | `CoreMemory` | 核心记忆（SQLite key-value 存储） |
| `petcore/internal/memory/long_term.go` | `LongTerm` | 长期记忆（FTS5 全文检索） |
| `petcore/internal/memory/extractor.go` | `Extractor` | 事实提取（LLM 调用 + 规则引擎） |

## 记忆工作流

```
用户说话
   │
   ▼
L1 短期 ← 直接追加到上下文
   │
   ├── AI 在回复中发现新信息
   │      │
   │      ├── 结构化信息（姓名/偏好）→ L2 核心（SQLite）
   │      │
   │      └── 事件性信息 → 生成摘要碎片 → L3 长期（FTS5）
   │
   └── 下次对话
          │
          ▼
      1. 读取 L2 核心 → 用户偏好
      2. 检索 L3 长期 → FTS5 关键词匹配
      3. 融合 → 构建 context → LLM 推理
```

## 事实提取策略

两种方式互补：
1. **LLM 主动提取**（推荐的）：LLM 通过 `remember` 工具写入新发现的事实
2. **后台异步提取**（补充）：对话结束后，Extractor 分析对话内容提取新事实

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 记住、记得 | `MemoryManager.Remember()` | 写入新事实 |
| 查询、回忆 | `MemoryManager.Recall()` | 检索匹配记忆 |
| 忘记、清除 | `MemoryManager.Forget()` | 删除特定记忆 |
| 查看记忆、历史 | `MemoryManager.List()` | 列出所有记忆 |
