# LLM 适配层

## 职责

抽象不同 LLM 提供商的 API 差异，提供统一的调用接口。采用 Provider 接口 + 注册表模式，支持运行时选择和切换。默认使用 OpenAI 兼容 API，预留本地 Ollama 接入能力。

> 参考 Reasonix `internal/provider` 和 NyaDeskPet `ProviderRegistry` 设计。

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/llm/provider.go` | `Provider` | Provider 接口（`Stream()` / `Chat()`） |
| `petcore/internal/llm/provider.go` | — | 注册表（`Register()` / `NewProvider()`） |
| `petcore/internal/llm/mock/mock.go` | `MockProvider` | Mock 实现，环境变量控制回复内容/延迟 |
| `petcore/internal/llm/openai/openai.go` | — | OpenAI 兼容 API 实现（支持 DeepSeek/硅基流动等第三方 API） |
| `petcore/internal/llm/openai/deepseek.go` | — | DeepSeek 便捷注册（默认 base_url + 模型名） |
| `petcore/internal/llm/ollama/ollama.go` | — | Ollama 本地实现 |

## Provider 接口

```go
type Provider interface {
    Name() string
    Model() string
    Stream(ctx context.Context, req Request) (<-chan Chunk, error)
    Chat(ctx context.Context, req Request) (Response, error)
}

type Request struct {
    Messages    []Message
    SystemPrompt string
    Tools       []Tool
    Temperature float64
    MaxTokens   int
}

type Chunk struct {
    Type     ChunkType    // Text / Reasoning / ToolCall / Usage / Done / Error
    Text     string
    ToolCall *ToolCall
    Usage    *Usage
}
```

## 注册表模式

```go
// 各 Provider 在 init() 中自注册
func init() {
    Register("openai", func(cfg map[string]any) (Provider, error) {
        // 支持所有 OpenAI 兼容 API（DeepSeek / 硅基流动 / Groq 等）
        // 通过 base_url 配置指向不同服务商
    })
}

// deepseek 是 openai 的便捷变体，默认指向 DeepSeek 官方 API
func init() {
    Register("deepseek", func(cfg map[string]any) (Provider, error) {
        // 默认 https://api.deepseek.com/v1，模型 deepseek-chat
    })
}

// main.go 中 blank import 触发注册
import (
    _ "petcore/internal/llm/openai"   // 注册 openai + deepseek
    _ "petcore/internal/llm/ollama"
)
```

## 第三方模型接入

所有兼容 OpenAI 格式的第三方模型**直接支持**，无需编写新 Provider。只需切换 `base_url` 和 `model`。

**配置文件示例（TOML）：**

```toml
# DeepSeek
[llm]
provider = "openai"
model = "deepseek-chat"
base_url = "https://api.deepseek.com/v1"
api_key_env = "DEEPSEEK_API_KEY"

# 或用便捷注册（效果同上）
# provider = "deepseek"
# model = "deepseek-chat"
# api_key_env = "DEEPSEEK_API_KEY"

# 硅基流动
# [llm]
# provider = "openai"
# model = "Qwen/Qwen2.5-7B-Instruct"
# base_url = "https://api.siliconflow.cn/v1"
# api_key_env = "SILICONFLOW_API_KEY"

# Groq
# [llm]
# provider = "openai"
# model = "llama-3.3-70b-versatile"
# base_url = "https://api.groq.com/openai/v1"
# api_key_env = "GROQ_API_KEY"
```

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 模型切换、API Key | `NewProvider()` | 通过配置选择 Provider |
| 对话、流式输出 | `Provider.Stream()` | 流式调用，逐 chunk 推送 |
| Mock 测试 | `MockProvider` | 环境变量 `MOCK_LLM_REPLY`/`_DELAY`/`_SHOW_REASONING` 控制行为 |
| Function Call | `Agent.Run()` | LLM 调用循环 + 工具执行 (Phase 4) |
| 本地模型 | `OllamaProvider` | Ollama 本地部署支持（⏳ 骨架） |
