# LLM 适配层

## 职责

抽象不同 LLM 提供商的 API 差异，提供统一的调用接口。采用 Provider 接口 + 注册表模式，支持运行时选择和切换。默认使用 OpenAI 兼容 API，预留本地 Ollama 接入能力。

> 参考 Reasonix `internal/provider` 和 NyaDeskPet `ProviderRegistry` 设计。

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/llm/provider.go` | `Provider` | Provider 接口（`Stream()` / `Chat()`） |
| `petcore/internal/llm/registry.go` | `Registry` | 注册表（`Register()` / `New()`） |
| `petcore/internal/llm/openai.go` | `OpenAIProvider` | OpenAI 兼容 API 实现 |
| `petcore/internal/llm/ollama.go` | `OllamaProvider` | Ollama 本地实现（预留） |

## Provider 接口

```go
type Provider interface {
    Name() string
    Model() string
    Stream(ctx context.Context, req Request) (<-chan Chunk, error)
    Chat(ctx context.Context, req Request) (Response, error)
}

type Request struct {
    Messages    []Message
    SystemPrompt string
    Tools       []Tool
    Temperature float64
    MaxTokens   int
}

type Chunk struct {
    Type     ChunkType    // Text / Reasoning / ToolCall / Usage / Done / Error
    Text     string
    ToolCall *ToolCall
    Usage    *Usage
}
```

## 注册表模式

```go
// 各 Provider 在 init() 中自注册
func init() {
    Register("openai", func(cfg Config) (Provider, error) {
        return NewOpenAIProvider(cfg)
    })
}

// main.go 中 blank import 触发注册
import (
    _ "petcore/internal/llm/openai"
    _ "petcore/internal/llm/ollama"
)
```

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 模型切换、API Key | `Registry.New()` | 通过配置选择 Provider |
| 对话、流式输出 | `Provider.Stream()` | 流式调用，逐 chunk 推送 |
| Function Call | `Agent.Run()` | LLM 调用循环 + 工具执行 |
| 本地模型 | `OllamaProvider` | Ollama 本地部署支持 |
