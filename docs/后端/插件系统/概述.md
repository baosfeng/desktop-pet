# 插件系统

## 职责

允许第三方开发者通过插件扩展宠物能力。采用三层架构（L1 YAML 动作包 / L2 JS 脚本 / L3 MCP 协议）。Phase 1 提供 Mock 实现和 Registry 接口，Phase 4 实现各层加载器。

## 三层架构

| 层级 | 技术 | 存储 | 热加载 | 阶段 |
|------|------|------|--------|------|
| L1 | YAML 动作包 | `~/.desktop-pet/actions/*.yaml` | fsnotify | Phase 4 |
| L2 | JS 脚本 (goja) | `~/.desktop-pet/scripts/*.js` | fsnotify | Phase 4 |
| L3 | MCP 子进程 (stdio JSON-RPC) | `config.toml` 配置 | 子进程管理 | Phase 4 |

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/plugin/registry.go` | `Plugin` | 插件接口（Name/Type/Start/Stop） |
| `petcore/internal/plugin/registry.go` | `Registry` | 插件注册中心（Register/List/Get/StartAll/StopAll） |
| `petcore/internal/plugin/registry.go` | `MockPlugin` | Mock 实现，供测试使用 |

## Plugin 接口

```go
type Plugin interface {
    Name() string
    Type() Type
    Start(ctx context.Context) error
    Stop() error
}
```

## 插件发现（Phase 4 实现）

```
启动时：
  1. 扫描 actions/ 目录 → 加载 *.yaml → L1
  2. 扫描 scripts/ 目录 → 加载 *.js  → L2
  3. 读取 config.toml → 启动 MCP 子进程 → L3

运行时：
  - fsnotify 监控 actions/ 和 scripts/ 目录
  - 文件变更 → 自动重载（无需重启）
  - MCP 子进程崩溃 → 自动重启
```

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 动作、表情、行为 | `Plugin` (Phase 4) | L1 YAML 动作包 |
| 脚本、自动化 | `Plugin` (Phase 4) | L2 JS 脚本 |
| MCP、工具、扩展 | `Plugin` (Phase 4) | L3 MCP 协议 |
| 热加载、热重载 | (Phase 4) | 文件监控自动重载 |
