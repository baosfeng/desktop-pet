# 插件系统

## 职责

允许第三方开发者通过插件扩展宠物能力。采用三层架构（L1 YAML 动作包 / L2 JS 脚本 / L3 MCP 协议），覆盖从声明式配置到完整工具扩展的不同复杂度需求。

> 参考 Espanso（YAML 热重载）、CopyQ（JS 脚本引擎）、Reasonix（MCP 子进程）设计。

## 三层架构

```
┌──────────────────────────────────────────────────────────┐
│  L1: 声明式动作包 (YAML)                                 │
│  配置驱动的动作/表情/行为组合，无需代码                     │
│  存储：~/.desktop-pet/actions/*.yaml                     │
│  热加载：fsnotify 文件监控自动重载                         │
├──────────────────────────────────────────────────────────┤
│  L2: 脚本插件 (JavaScript)                               │
│  轻量级脚本扩展，可编程但安全                              │
│  存储：~/.desktop-pet/scripts/*.js                       │
│  引擎：goja (纯 Go JS 引擎，无需 V8)                      │
│  沙箱：限制文件/网络访问                                   │
├──────────────────────────────────────────────────────────┤
│  L3: MCP 工具/外部进程                                    │
│  标准 MCP 协议，子进程隔离运行                              │
│  协议：stdio JSON-RPC                                    │
│  沙箱：独立进程 + 权限声明                                 │
└──────────────────────────────────────────────────────────┘
```

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/plugin/registry.go` | `Registry` | 插件注册中心 |
| `petcore/internal/plugin/loader.go` | `Loader` | 文件监控 + 热加载 (fsnotify) |
| `petcore/internal/plugin/l1_yaml.go` | `YAMLLoader` | L1 YAML 动作包加载器 |
| `petcore/internal/plugin/l2_script.go` | `ScriptEngine` | L2 JS 脚本引擎 (goja) |
| `petcore/internal/plugin/mcp_client.go` | `MCPClient` | L3 MCP 客户端 |
| `petcore/internal/plugin/sandbox.go` | `Sandbox` | 沙箱执行环境 |

## 插件发现

```
启动时：
  1. 扫描 actions/ 目录 → 加载 *.yaml → L1
  2. 扫描 scripts/ 目录 → 加载 *.js  → L2
  3. 读取 config.toml → 启动 MCP 子进程 → L3

运行时：
  - fsnotify 监控 actions/ 和 scripts/ 目录
  - 文件变更 → 自动重载（无需重启）
  - MCP 子进程崩溃 → 自动重启
```

## 业务场景映射

| 需求关键词 | 入口 | 说明 |
|-----------|------|------|
| 动作、表情、行为 | `YAMLLoader` | L1 YAML 动作包 |
| 脚本、自动化 | `ScriptEngine` | L2 JS 脚本 |
| MCP、工具、扩展 | `MCPClient` | L3 MCP 协议 |
| 热加载、热重载 | `Loader` | 文件监控自动重载 |
