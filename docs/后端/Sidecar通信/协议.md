# Sidecar 通信协议规范

> Tauri Rust 壳 ↔ Go PetCore 之间的通信协议。
> 传输层：stdin/stdout JSON 行（Tauri sidecar 原生支持）

---

## 1. 协议概述

| 属性 | 值 |
|------|-----|
| 传输 | stdin/stdout（Tauri sidecar 管理子进程） |
| 编码 | UTF-8 JSON，每行一条消息，`\n` 分隔 |
| 双工 | 全双工（Tauri 写入 stdin，PetCore 写入 stdout） |
| 版本 | 1.0 |
| 错误处理 | JSON 解析失败时返回 `{"type":"error","code":"PARSE_ERROR"}` |

## 2. 消息类型

### 2.1 请求（Tauri → PetCore, stdin）

```json
{
  "type": "cmd",
  "id": "1",
  "method": "chat",
  "params": {
    "text": "你好"
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` | ✅ | 固定为 `"cmd"` |
| `id` | `string` | ✅ | 请求标识符，响应中回显 |
| `method` | `string` | ✅ | 方法名（见 §3） |
| `params` | `object` | ✅ | 方法参数 |

### 2.2 响应（PetCore → Tauri, stdout）

```json
{
  "type": "resp",
  "id": "1",
  "result": {
    "done": true
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` | ✅ | 固定为 `"resp"` |
| `id` | `string` | ✅ | 回显请求的 `id` |
| `result` | `any` | ✅ | 方法返回值 |

### 2.3 事件推送（PetCore → Tauri, stdout）

```json
{
  "type": "event",
  "event": "pet.speak",
  "data": {
    "text": "你好~",
    "emotion": "happy"
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` | ✅ | 固定为 `"event"` |
| `event` | `string` | ✅ | 事件类型（见 §4） |
| `data` | `object` | ✅ | 事件负载 |

### 2.4 错误（任意方向）

```json
{
  "type": "error",
  "id": "2",
  "code": "NOT_READY",
  "message": "pet not initialized"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` | ✅ | 固定为 `"error"` |
| `id` | `string` | ❌ | 对应请求的 `id`（请求错误时必填） |
| `code` | `string` | ✅ | 错误码 |
| `message` | `string` | ✅ | 可读描述 |

## 3. 方法列表

| 方法 | 说明 | Phase |
|------|------|-------|
| `ping` | 健康检查，应返回 `{"pong": true}` | 1 |
| `chat` | 发送对话消息 | 1 |
| `get_status` | 获取宠物当前状态 | 1 |
| `get_config` | 获取配置 | 1 |
| `set_config` | 修改配置 | 1 |
| `query_memory` | 查询记忆 | 1 |
| `list_plugins` | 列出已加载插件 | 2 |
| `reload_plugin` | 重载指定插件 | 2 |
| `reset_memory` | 重置记忆 | 2 |
| `shutdown` | 优雅关闭 | 1 |

### 3.1 `chat` 方法

```json
// 请求
{"type":"cmd","id":"1","method":"chat","params":{"text":"你好","session_id":"..."}}

// 响应（最终）
{"type":"resp","id":"1","result":{"done":true}}

// 流式事件（chat 过程中推送）
{"type":"event","event":"agent.reply","data":{"text":"你好","done":false}}
{"type":"event","event":"agent.reply","data":{"text":"~","done":false}}
{"type":"event","event":"agent.reply","data":{"text":" 我是小宠物！","done":true}}
```

### 3.2 `ping` 方法

```json
// 请求
{"type":"cmd","id":"3","method":"ping","params":{}}

// 响应
{"type":"resp","id":"3","result":{"pong":true,"uptime":3600}}
```

## 4. 事件类型

| 事件 | 方向 | 触发时机 |
|------|------|---------|
| `state.changed` | PetCore → 前端 | 状态机切换（待机→关注→交互→说话） |
| `pet.speak` | PetCore → 前端 | 宠物说话 |
| `pet.action` | PetCore → 前端 | 播放 Live2D 动画 |
| `pet.emotion` | PetCore → 前端 | 情绪变化 |
| `agent.thinking` | PetCore → 前端 | AI 思考中/完成 |
| `agent.reply` | PetCore → 前端 | 流式回复片段 |
| `memory.updated` | PetCore → 前端 | 记忆更新通知 |
| `error` | PetCore → 前端 | 错误通知 |

### 4.1 `state.changed`

```json
{
  "type": "event",
  "event": "state.changed",
  "data": {
    "from": "idle",
    "to": "interacting"
  }
}
```

状态映射：`idle` / `attentive` / `interacting` / `speaking`

### 4.2 `agent.reply`（流式）

```json
{"type":"event","event":"agent.reply","data":{"text":"你好","done":false}}
```

`done: true` 表示流结束。

## 5. 生命周期

```
Tauri 启动
  │
  ├── spawn PetCore 子进程 (sidecar)
  ├── 等待 ready 事件（超时 10 秒）
  │
  ├── [循环] 健康检查 (每 5 秒 ping)
  ├── [循环] 事件转发 (stdout → Tauri event)
  │
  ├── 子进程崩溃 → 自动重启（最多 3 次）
  │
  └── Tauri 关闭
      ├── 发送 shutdown 命令
      ├── 等待优雅退出（超时 5 秒）
      └── 强制 kill
```

## 6. 错误码

| 错误码 | 说明 |
|--------|------|
| `PARSE_ERROR` | JSON 解析失败 |
| `NOT_READY` | PetCore 尚未就绪 |
| `UNKNOWN_METHOD` | 未知方法名 |
| `INVALID_PARAMS` | 参数校验失败 |
| `INTERNAL_ERROR` | 内部错误 |
| `TIMEOUT` | 操作超时 |
| `SHUTTING_DOWN` | 正在关闭 |
