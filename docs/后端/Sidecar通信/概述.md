# Sidecar 通信

## 职责

管理 Tauri Rust 壳与 Go PetCore 子进程之间的通信。Tauri 通过 sidecar 模式启动 PetCore，双方通过 stdin/stdout JSON 行进行双向通信。

## 通信架构

```
前端 (React WebView)
   │ Tauri IPC (invoke + event)
   ▼
Tauri Rust 壳 (src-tauri)
   │ stdin/stdout JSON (sidecar)
   ▼
Go PetCore (petcore)
```

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/server/transport.go` | `Transport` | 传输层接口 |
| `petcore/internal/server/stdio.go` | `StdioTransport` | stdin/stdout JSON 传输 |
| `src-tauri/src/sidecar.rs` | `SidecarManager` | Tauri 端 sidecar 生命周期管理 |

## 通信协议

PetCore 启动后通过 stdin 读取命令，通过 stdout 输出事件：

```
→ Tauri 写入 stdin (JSON 行):
{"type":"cmd","id":"1","method":"chat","params":{"text":"你好"}}
{"type":"cmd","id":"2","method":"get_status","params":{}}

← PetCore 输出 stdout (JSON 行):
{"type":"event","event":"pet.speak","data":{"text":"你好~","emotion":"happy"}}
{"type":"event","event":"pet.action","data":{"action":"wave","duration":2000}}
{"type":"resp","id":"1","result":{"done":true}}
```

## 消息类型

| 方向 | Type | 说明 |
|------|------|------|
| Tauri → PetCore | `cmd` | 命令请求，包含 method 和 params |
| PetCore → Tauri | `event` | 事件推送（状态变化、说话、动作等） |
| PetCore → Tauri | `resp` | 命令响应，对应 cmd 的 id |
| PetCore → Tauri | `error` | 错误信息 |

## 生命周期管理

```
Tauri 启动
   │
   ├── 从 binaries/ 找到 petcore 二进制
   ├── spawn 子进程（stdin/stdout pipe）
   │
   ├── 发送 "ping" 心跳检测（每 5 秒）
   ├── 子进程崩溃 → 自动重启
   │
   └── Tauri 关闭 → 发送 shutdown → 等待退出
```
