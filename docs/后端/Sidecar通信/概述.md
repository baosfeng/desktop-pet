# Sidecar 通信

## 职责

管理 Tauri Rust 壳与 Go PetCore 子进程之间的通信。Tauri 通过 sidecar 模式启动 PetCore，双方通过 stdin/stdout JSON 行进行双向通信。

## 通信架构

```
前端 (React WebView)
   │ Tauri IPC (invoke + event)
   ▼
Tauri Rust 壳 (src-tauri)
   │ stdin/stdout JSON (sidecar)
   ▼
Go PetCore (petcore)
```

## 核心类/接口

| 文件路径 | 类/接口 | 职责 |
|---------|---------|------|
| `petcore/internal/server/server.go` | `Server` | sidecar 通信服务，读取 stdin 命令并处理 |
| `petcore/internal/server/server.go` | `SinkAdapter` | 将 Server 输出适配为 event.Sink 接口 |
| `src-tauri/src/sidecar.rs` | `SidecarWriter` | Tauri 端，向 sidecar stdin 写入 JSON |
| `src-tauri/src/sidecar.rs` | `start_sidecar()` | 启动子进程、事件转发、自动重启（最多 3 次） |

详见 [通信协议](协议.md)

## 生命周期管理

```
Tauri 启动
   │
   ├── 从 binaries/ 找到 petcore 二进制
   ├── spawn 子进程（stdin/stdout pipe）
   ├── 发送 ping 健康检查
   │
   ├── [goroutine] 读取 stdout → 解析 JSON → app.emit() 推给前端
   ├── 子进程崩溃 → 自动重启（最多 3 次）
   │
   └── Tauri 关闭 → 子进程退出
```
