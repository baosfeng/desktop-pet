# 模块规划：事件系统 (petcore/internal/event)

> **前置依赖：** 无（零内部依赖）
> **被以下模块依赖：** fsm、core、server（以及所有 Shell 层）
> **对应 Phase：** Phase 1 — MVP

---

## 1. 模块定位

事件系统是 PetCore 最基础的模块——所有模块间通信、Shell 输出都通过事件来传递。它的职责是定义"PetCore 里有哪些事件"以及"如何消费事件"。

```
┌──────────────┐    Sink 接口     ┌──────────────────┐
│  PetCore 内核 │ ─────────────→  │  Shell 层         │
│  (core/fsm/   │   Send(Event)   │  (Tauri / CLI /   │
│   agent/...)  │                 │   Server)         │
└──────────────┘                 └──────────────────┘
```

## 2. 接口与边界

### 对外暴露

```go
// Sink — 所有 Shell 必须实现的接口
type Sink interface {
    Send(event Event) error
    Close() error
}

// Event — 跨层传递的标准事件
type Event struct {
    Kind Type
    Data any
    Meta Meta
}

// Type — 8 种事件类型
const (
    EventStateChanged  Type = "state.changed"
    EventPetSpeak      Type = "pet.speak"
    EventPetAction     Type = "pet.action"
    EventPetEmotion    Type = "pet.emotion"
    EventAgentThinking Type = "agent.thinking"
    EventAgentReply    Type = "agent.reply"
    EventMemoryUpdated Type = "memory.updated"
    EventError         Type = "error"
)

// Meta — 事件元信息
type Meta struct {
    Timestamp int64  // Unix 毫秒
    Source    string // core / plugin / user
    SessionID string
}
```

### 依赖的外部接口

- 无（只依赖 Go 标准库 `time`）

## 3. 可拆卸性设计

| 机制 | 实现 |
|------|------|
| **零内部依赖** | `import` 只有 `"time"`，可以被任何 Go 项目独立编译 |
| **Shell 解耦** | `Sink` 接口是 PetCore 与外界唯一的输出通道 |
| **测试替身** | `NoopSink` 内置空实现；测试可用 channel-backed 自定义 Sink |
| **编译时检查** | `var _ Sink = (*NoopSink)(nil)` |
