# 模块规划：配置管理 (petcore/internal/config)

> **前置依赖：** 无（零内部依赖）
> **被以下模块依赖：** core、memory（通过 Config 引用）
> **对应 Phase：** Phase 1 — MVP

---

## 1. 模块定位

配置管理是 PetCore 的基础模块，负责加载、校验和提供配置数据。所有模块的行为都由配置驱动。

## 2. 接口与边界

### 对外暴露

```go
func Load(path string) (*Config, error)    // 从 TOML 文件加载
func DefaultConfig() *Config               // 返回默认配置（每次独立副本）

// API Key 从环境变量读取
func (c *LLMConfig) APIKey() string
```

### Config 结构体

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `LLM.Provider` | string | `"mock"` | openai / ollama / mock |
| `LLM.Model` | string | `"mock-v1"` | 模型名称 |
| `LLM.BaseURL` | string | `""` | API 地址 |
| `LLM.APIKeyEnv` | string | `""` | 环境变量名 |
| `Agent.SystemPrompt` | string | 英文占位 | 系统提示词 |
| `Memory.ShortTermSize` | int | `20` | L1 短期轮数 |
| `Window.AlwaysOnTop` | bool | `true` | 窗口置顶 |
| `Update.AutoCheck` | bool | `true` | 自动检查更新 |
| `Plugin.ActionsDir` | string | `~/.desktop-pet/actions` | L1 YAML 目录 |

### 依赖的外部接口

- 无（只依赖 Go 标准库 + TOML 解析器）

## 3. 可拆卸性设计

| 机制 | 实现 |
|------|------|
| **零内部依赖** | 只依赖标准库 `os`、`path/filepath` |
| **独立测试** | 用 `t.TempDir()` 创建临时配置文件 |
| **防御性复制** | `DefaultConfig()` 每次返回新实例 |
| **环境变量解耦** | API Key 通过环境变量读取，不硬编码 |
