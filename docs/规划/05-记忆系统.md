# 模块规划：记忆系统 (petcore/internal/memory)

> **前置依赖：** 无（不依赖其他 PetCore 内部包）
> **被以下模块依赖：** agent（通过 Manager 接口）、core（通过 Manager 接口）
> **对应 Phase：** Phase 1 — MVP

---

## 1. 模块定位

记忆系统是宠物的"大脑"，负责存储和检索对话上下文、用户事实和历史事件。三层架构提供不同的持久性和检索能力。

## 2. 接口与边界

### 对外暴露

```go
type Manager interface {
    // L1 短期记忆（内存环形缓冲区）
    AddShortTerm(msg Message)
    GetShortTerm() []Message
    ClearShortTerm()

    // L2 核心记忆（SQLite key-value，Phase 2 落地）
    Remember(key, value string, importance int) error
    Recall(key string) (string, error)
    GetAllCore() (map[string]string, error)

    // L3 长期记忆（SQLite + FTS5，Phase 2 落地）
    Store(ctx context.Context, fact Fact) error
    Search(ctx context.Context, query string, limit int) ([]Fact, error)
}

type Message struct {
    Role    string   // system / user / assistant / tool
    Content string
}

type Fact struct {
    Key        string
    Value      string
    Category   string   // preference / fact / event
    Importance int       // 1-5
    Timestamp  int64
}
```

### 对外提供

| 用途 | 实现 | 说明 |
|------|------|------|
| **Mock** | `NewMockManager()` | 返回空的 `InMemoryManager` |
| **生产** | `NewInMemoryManager()` | 全内存实现，Phase 1 可用；Phase 2 替换为 SQLite |

## 3. 可拆卸性设计

| 机制 | 实现 |
|------|------|
| **接口统一** | 所有三层操作通过 `Manager` 接口，底层替换不影响使用者 |
| **Mock 内置** | `NewMockManager()` 同包提供 |
| **并发安全** | `sync.RWMutex` 保护所有操作，测试中验证并发安全 |
| **Phase 渐进** | Phase 1 全内存，Phase 2 无缝切换到 SQLite |
| **编译时检查** | `var _ Manager = (*InMemoryManager)(nil)` |
