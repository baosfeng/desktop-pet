# 模块规划：状态机 (petcore/internal/fsm)

> **前置依赖：** event（仅依赖事件类型常量）
> **被以下模块依赖：** core（通过 Machine 接口）
> **对应 Phase：** Phase 1 — MVP

---

## 1. 模块定位

FSM 是宠物行为状态机——纯逻辑模块，定义宠物在 4 个状态之间的转换规则。它不关心窗口行为、不关心 AI 对话，只回答"当前状态是什么"和"从这个状态可以转换到哪个状态"。

```
状态转换表（硬编码，不可热加载）：
  idle ──(detect_user)──→ attention
  attention ──(user_input)──→ interaction
  interaction ──(agent_reply)──→ speaking
  speaking ──(done)──→ idle
  attention ──(timeout)──→ idle
  interaction ──(error)──→ idle
```

## 2. 接口与边界

### 对外暴露

```go
type Machine interface {
    Current() State                                     // 当前状态
    Transition(evt event.Type) error                    // 按事件转换
    OnTransition(fn func(from, to State))               // 注册转换回调
}

type State string
const (
    StateIdle        State = "idle"
    StateAttention   State = "attention"
    StateInteraction State = "interaction"
    StateSpeaking    State = "speaking"
)

func IsValidTransition(from State, evt event.Type) bool  // 检查转换合法性
func TransitionsFrom(s State) []event.Type               // 查询可用事件
```

### 依赖的外部接口

- `event.Type` — 仅用到事件类型常量，不依赖 event 包其他内容

### 对外提供

| 用途 | 实现 | 说明 |
|------|------|------|
| **Mock** | `fsm.NewMockMachine(initial State)` | 返回 MockMachine，所有 Transition 都是 no-op，供其他模块测试使用 |

## 3. 可拆卸性设计

| 机制 | 实现 |
|------|------|
| **低依赖** | 仅依赖 `event`（同一基础层） |
| **纯逻辑** | 无 IO、无 goroutine、无外部状态 |
| **Mock 实现** | `MockMachine` 同包提供，Transition 为 no-op |
| **回调解耦** | `OnTransition` 注册回调，不直接调用窗口/网络 |
| **表格驱动测试** | 13 条转换规则全覆盖 |
| **编译时检查** | `var _ Machine = (*MockMachine)(nil)` |
