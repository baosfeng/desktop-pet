# 模块规划：工具系统 + 插件系统 (petcore/internal/tool + plugin)

> **前置依赖：** tool — 无；plugin — 无
> **被以下模块依赖：** agent（依赖 tool.Registry）、core（依赖两者）
> **对应 Phase：** Phase 1 — MVP（骨架）→ Phase 4 — 完整实现

---

## 1. 模块定位

工具系统和插件系统是 PetCore 的扩展能力入口：

- **工具系统** (`tool`)：LLM 可调用的能力单元（web_search、remember、desktop_ctrl 等）。注册中心模式，内置工具和 MCP 工具统一注册。
- **插件系统** (`plugin`)：三层插件架构（L1 YAML 动作包 / L2 JS 脚本 / L3 MCP 子进程）。管理插件的注册、发现和生命周期。

两者共享相同的注册中心模式，Phase 1 仅提供骨架和 Mock。

## 2. 接口与边界

### 工具系统

```go
type Tool interface {
    Name() string
    Description() string
    Execute(ctx context.Context, args string) (string, error)
}

type Registry interface {
    Register(tool Tool) error
    Execute(ctx context.Context, name, args string) (string, error)
    List() []Tool
}
```

### 插件系统

```go
type Plugin interface {
    Name() string
    Type() Type         // L1/YAML / L2/JS / L3/MCP
    Start(ctx context.Context) error
    Stop() error
}

type Registry interface {
    Register(p Plugin) error
    List() []Plugin
    Get(name string) (Plugin, error)
    StartAll(ctx context.Context) error
    StopAll() error
}
```

## 3. 可拆卸性设计

| 机制 | 实现 |
|------|------|
| **注册中心模式** | 新增 tool/plugin 只需调用 `Register()` |
| **接口边界** | Tool/Plugin 都是接口，可独立实现 |
| **Mock 内置** | MockTool 和 MockPlugin 同包提供 |
| **并发安全** | `sync.RWMutex` 保护注册表 |
| **编译时检查** | `var _ Registry = (*builtinRegistry)(nil)` |
