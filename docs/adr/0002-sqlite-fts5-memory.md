# ADR 0002: 使用 SQLite + FTS5 作为记忆存储引擎

- **日期**: 2025-07
- **状态**: ✅ 已采纳

## 背景

宠物需要记忆系统来存储用户信息、对话历史和事实。需要满足：本地优先、零依赖、支持全文检索。

## 决策

使用 SQLite + FTS5（通过 [modernc.org/sqlite](https://modernc.org/sqlite) 纯 Go 实现）作为记忆存储引擎。

三层记忆架构：
- **L1 短期**: 内存环形缓冲区（最近 20 轮对话）
- **L2 核心**: SQLite key-value（用户偏好/结构化事实）
- **L3 长期**: SQLite + FTS5（对话摘要/历史事件，支持全文检索）

## 备选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **SQLite + FTS5** (选定) | 纯 Go 零 CGO, 零运维, FTS5 内置 | 无向量检索 |
| ChromaDB | 向量检索原生支持 | 额外进程, 运维复杂 |
| Qdrant | 高性能向量检索 | 独立服务, 资源开销大 |

## 影响

- Phase 1 先用 FTS5 实现基础检索
- Phase 2 按需接入嵌入式向量库（如 `weaviate` 或自建简单索引）
- 所有用户数据存储在 `~/.desktop-pet/`，独立于 .app 之外
