# ADR 0003: 三层插件系统：YAML / JS / MCP

- **日期**: 2025-07
- **状态**: ✅ 已采纳

## 背景

宠物需要有灵活的扩展能力，让不同技术水平的用户都能为宠物添加自定义行为。

## 决策

采用三层插件架构：

| 层级 | 技术 | 用户 | 热加载 |
|------|------|------|--------|
| L1 | YAML 动作包（声明式配置） | 非开发者 | fsnotify 文件监控 |
| L2 | JS 脚本（goja 引擎） | 进阶用户 | fsnotify 文件监控 |
| L3 | MCP 子进程（stdio JSON-RPC） | 开发者/生态 | 独立进程管理 |

## 备选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **三层架构** (选定) | 覆盖全用户层级 | 实现复杂度较高 |
| 仅 L3 MCP | 生态兼容性好 | 普通用户无法使用 |
| 仅 L1 YAML | 简单易用 | 表达能力有限 |

## 影响

- L1 继承自 Espanso 的 YAML 包管理器设计
- L2 使用 goja（纯 Go JS 引擎，无需 V8），沙箱限制文件/网络访问
- L3 使用标准 MCP 协议，兼容未来生态工具
- 所有插件目录通过 fsnotify 监控，文件变更自动重载无需重启
